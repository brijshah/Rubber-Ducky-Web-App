Payload Generation Tips

In order for the scripts from the payload generator to run, the 'victim' machine must have administrator/sudo privilages.
Selecting Scripts
At this time, it is recommended that you only select either a reconnaisance script or an exploit script. The reconnaisance scripts require a reporting script to be selected along with it whereas the exploit ones do not. Selecting either script is enforced in order to keep the ducky from being exposed. The longer they take to run, the higher the chance of exposing it to the user. Failing to adhere to these rules will result in an error message to be displayed on the web page with the appropriate message.
Downloading Script
Once you have selected the required scripts, the download button will appear allowing you to retrieve both the text file containing the generated script as well as the bin file ready to be deployed.
Running Script
  • Copy bin file to SD Card
  • Insert SD Card into USB Rubber Ducky
  • Insert USB Rubber Ducky into 'victim' machine

DuckyScript
The Ducky Script is the language of the USB Rubber Ducky. Writing Scripts can be done using the Encoder web page.
Syntax

Ducky Syntax is simple. Each command resides on a new line and may have options follow them. The commands are all written in CAPS. Most of the commands will invoke keystrokes or a combination of keys, while some offer delays or pauses. Below are the key commands you can use to create your scripts.

REM: Lines that begin with REM will not be processed. REM is a comment.

DEFAULT_DELAY: is used to define how long in milliseconds to wait before next command is executed.

DELAY: creates a momentary pause in the script. Used for creating a momentary pause between sequential commands that may take the 'victim' machine some time to process.

STRING: processes the text following taking special care to auto-shift.

WINDOWS or GUI: used to emulate the Windows key or CMD key.
